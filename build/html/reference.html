Code Reference¶

Here you will find reference to the specific functions used in the RAFIKI pacakge. These are found in two python files: Generating_SZ_Data.py which contains the methods for cutting particles, and creating a projected map of the tSZ signal in your box, and functions.py which contains all the functions relevant for analysis. Below we summarize the role of each function as well as its inputs and outputs.

Generating_SZ_Data.generating_sz_data(filename, projection_direction, output_name, frb, redshift)¶

Cuts ISM and current wind particles, makes a new field of gas pressure, projects this field in the specified direction, and saves a file containing the projected map of the tSZ signal

Parameters

  • filename (str) – path to snapshot

  • projection_direction (str) – ‘x’, ‘y’, or ‘z’, the direction you want to project your data in

  • output_name (str) – name of the .npy file that will store the projected SZ data

  • frb (int) – number of pixels in your fixed resolution buffer. Suggested to correspond to twice the resolution of your observational comp

  • redshift (int) – redshift of the snapshot

Returns

None. Saves a file with the name given in the inputs containing the projected SZ data

functions.sorting(attribute, minimum)¶

Identifies galaxies by a specified attribute that are above the minimum value for that attribute. Used to cut low-mass galaxies to speed up the analysis process

Parameters

  • attribute (list[float]) – list of galaxy stellar masses (can be altered to other galaxy properties if desired)

  • mininimum – minimum value you want to allow in the sample

Returns

gals_index-index of the galaxies in Caesar catalog that are above the set minimum

Return type

list[int]

functions.cropping_sz_x(sz_dat, xs, ys, zs, sample, width, frb)¶

Generates a nested 2D array of SZ data in the regions around our sampled galaxies using a data box projected in the x-direction

Parameters

  • sz_dat (list[float]) – 2D array of projected Compton-y map

  • xs (list[float]) – x-coordinates of locations of galaxies in your box, adjusted to match the pixel frb

  • ys (list[float]) – y-coordinates of locations of galaxies in your box, adjusted to match the pixel frb

  • zs (list[float]) – z-coordinates of locations of galaxies in your box, adjuste to match the pixel frb

  • sample (list[int]) – list of index locations of the galaxies we want to sample (output of sorting)

  • width (int) – the pixel size we want to crop around the galaxies

  • frb (int) – pixel size of the SZ data used

Returns

lists-Nested array of SZ data for the regions around our sample gaalxies

Return type

list[float]

functions.azimuthalAverage(image, center=None)¶

Calculates the azimuthally averaged radial profile. Taken from with some alterations https://github.com/mkolopanis/python/blob/master/radialProfile.py

Parameters

  • image (list[float]) – 2D array of projected Compton-y map around our galaxy of interest

  • center (list[float] or None) – The [x,y] pixel coordinates used as the center. Default is None, which useds the center of the image

Returns

radial_prof-Radial profile of the azimuthally averaged signal

Return type

list[float]

functions.make_radial_profiles(stamps, kernel, label)¶

Convolves data and generates radial profiles for all galaxies in your sample

Parameters

  • stamps (list[float]) – A nested array of the SZ data around galaxies. Output of cropping_sz function

  • kernel (float) – The standard deviation of the Gaussian kernel in units of pixels

  • label (str) – Label you want in the file name containing radial profiles

Returns

Makes a csv file entitled radial_label.csv containing radial profiles around each galaxy in your sample

The next function is used when calculating thermal energy

functions.thermal_energy(stamps, kernel, label, aperature)¶

Calculates the thermal energy in a given aperature from maps of SZ-y data

Parameters

  • stamps (list[float]) – A nested array of the SZ data around galaxies. Output of cropping_sz function

  • kernel (float) – The standard deviation of the Gaussian kernel in units of pixels

  • label (str) – Label you want in the file name containing thermal energy data

  • aperature (float) – pixel size of the radius of the aperature within which you want to caculate thermal energy

Returns

Saves a csv file containing the thermal energy around each of the galaxies in your sample

The next set of functions are used when calculating error via the bootstrapping method

functions.gen_random_indices(index_set, gen_size)¶

Generates a list of indicies by random sampling with replacement

Parameters

  • index_set (list[float]) – List of values to sample from

  • gen_size (int) – Length of final resampled list you want

Returns

a list of length gen_size randomly chosen from index_set

functions.single_catalog_bootstrap(data, boot_size, loop_size)¶

Calculates the means of a list of catalogs, useful when determining things like correlation matrices

Parameters

  • data (list[float]) – List of pandas tables

  • boot_size (int) – Sample size

  • loop_size – How many times to do the bootstrapping

The last function is used for plotting and visualizing data
